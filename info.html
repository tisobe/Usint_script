Obscat Data Help Page

  • If a USINT field is in black, the field is not editable by USINT.
  • If a USINT field is in purple, the field is editable by USINT.
  • If a USINT field is in red, the field is editable, but a CDO permission is needed to change.

Subarray Type subarray (acisparam) A subarray is a reduced region of the CCDs (all of the CCDs that are turned on) that will be read. A reduced region may also help to reduce the effects of pulse pile-up. The first box indicates whether the proposer intends to use the subarray capability. The default is 'None'. If CUSTOM is selected, the next three entires (Start, Rows, Frame Time) must be filled. Note that the minimum Frame Time depends on the size and location of the subarray
If CUSTOM is selected, Start, and Rows must be filled.
Value Ranges:NONE, 1/2, 1/4, 1/8 and CUSTOM
Start subarray_start_row (acisparam) Subarray: The starting row that will be read for processing custom subarrays. If it is not NULL, Subarray Type must be CUSTOM, and Rows must be filled.
Value Ranges: NULL, 1 to 924
Rows subarray_row_count (acisparam) Subarray: The number of rows that will be read for processing custom subarrays. If it is not NULL, Subarray Type must be CUSTOM, and Start must be filled. Value Ranges: NULL, 101 to 1024
Frame Time subarray_frame_time (acisparam) Subarray - Frame Time: The Frame Time is the fundamental unit of exposure for ACIS. The higher time resolution is achieved by reading fewer rows of the CCD. The minimum frame time of 0.2 sec corresponds to reading a subarray of 128 rows. The maximum time of 3.2 sec corresponds to reading all 1024 rows. Any value from 0.2 to 3.2 sec is possible by adjusting the number of rows to be read. The number of rows to be read and the Subarray Frame Time should be equivalent; the two values will be compared to see that they are. The equation for calculating the Frame Time from the number of rows to be read is included in the Proposer's Guide.
Value Ranges: NULL, 0 to 10
Duty Cycle duty_cycle (acisparam) Alternating Exposure Readout: Logical value indicating use of alternating exposure readout. The default is 'N'. Alternating Exposure Readout observation sets the number of SECONDARY exposures that will follow each primary exposure. A deadtime will result from the short exposure since the electronics still require 3.2 sec to process the full frame. Therefore, to minimize the deadtime, the number of short exposures should be kept to a minimum. If Y is selected, Tprimary and Tsecondary must be filled.
Value Ranges: NULL, Y, N
Number secondary_exp_count (acisparam) Alternating Exposure Readout: The number of secondary exposures that will follow each primary exposure. If n = 0, only primary exposures are used. Read the ACIS chapter of the Proposer's Guide for an estimate of the efficiency. The recommended value is 2.
Value Ranges: 0 to 15
Tprimary primary_exp_time (acisparam) Alternating Exposure Readout: Exposure Time for Primary Cycle: The primary exposure time in tenths of seconds. The recommended time for a non-zero number of secondary exposures is 0.3.
Value Ranges: NULL, 0 to 10
Tsecondary secondary_exp_time (acisparam) Alternating Exposure Readout : Exposure Time for Secondary Cycle: The secondary exposure time in tenths of seconds. The recommended time for a non-zero number of secondary exposures is 3.2
Onchip Summing onchip_sum (acisparam) Logical value indicating on-chip summation. The default value is 'N'. On-chip summation can be used to reduce the number of items per CCD readout. The spatial resolution is degraded as is the event splitting information. Currently, only 2x summation is supported. You can optionally fill Rows and Column.
Value Ranges: NULL, N, Y
Rows onchip_row_count (acisparam) # of rows for On-Chip Summing.
Value Ranges: 1 to 512
Columns onchip_column_count (acisparam) # of columns for On-Chip Summing.
Value Ranges: 1 to 512
Event Filter eventfilter (acisparam) This logical value indicates that the user wishes to filter every candidate event before packing into the telemetry stream. The filter applies to all of the active CCDs. The use of an event filter does NOT affect pulse pileup, but only reduces the telemetry. If Y is selected, Lower and Range must be filled.
Value Ranges: NULL, Y, N
Lower eventfilter_lower (acisparam) Event Filter: Lower Event Threshold: The value of the threshold that will be applied. Units are keV. If it is not NULL, Event filter must be Y.
Value Ranges: NULL, 0.1 to 15
Range eventfilter_higher (acisparam) Event Filter: Event Filter Range: The range of the event amplitude in energy channels. If it is not NULL, Event filter must be Y.
Value Ranges: 0.1 to 15
Window Filter spwindow (acisparam) This set of parameters allows the user to filter the data to reduce the telemetry load. The use of a Spatial Window does NOT affect the reading of the CCD, so there will be no impact on pulse pile-up. Spatial windows do reduce the telemetry by selecting photons to insert into the telemetry stream. Spatial Windows can either INclude all of the photons in the defined region or Exclude all of the photons in the defined region. If Y is selected, Chip, Photon Inclusion, Start Row, Start Column, Height, Width, Lower Energy, Energy Range, Sample Rate, Bias, Frequency Bias After.
Value Ranges: NULL, Y, N
